FramePayload

public enum FramePayload

Frame-type-specific payload data.

  • data(_:)

    A DATA frame, containing raw bytes.

    See RFC 7540 § 6.1.

    Declaration

    Swift

    indirect case data(FramePayload.Data)
  • headers(_:)

    A HEADERS frame, containing all headers or trailers associated with a request or response.

    Note that swift-nio-http2 automatically coalesces HEADERS and CONTINUATION frames into a single FramePayload.headers instance.

    See RFC 7540 § 6.2.

    Declaration

    Swift

    indirect case headers(Headers)
  • priority(_:)

    A PRIORITY frame, used to change priority and dependency ordering among streams.

    See RFC 7540 § 6.3.

    Declaration

    Swift

    case priority(StreamPriorityData)
  • rstStream(_:)

    A RST_STREAM (reset stream) frame, sent when a stream has encountered an error condition and needs to be terminated as a result.

    See RFC 7540 § 6.4.

    Declaration

    Swift

    case rstStream(HTTP2ErrorCode)
  • settings(_:)

    A SETTINGS frame, containing various connection–level settings and their desired values.

    See RFC 7540 § 6.5.

    Declaration

    Swift

    case settings(Settings)
  • pushPromise(_:)

    A PUSH_PROMISE frame, used to notify a peer in advance of streams that a sender intends to initiate. It performs much like a request’s HEADERS frame, informing a peer that the response for a theoretical request like the one in the promise will arrive on a new stream.

    As with the HEADERS frame, swift-nio-http2 will coalesce an initial PUSH_PROMISE frame with any CONTINUATION frames that follow, emitting a single FramePayload.pushPromise instance for the complete set.

    See RFC 7540 § 6.6.

    For more information on server push in HTTP/2, see RFC 7540 § 8.2.

    Declaration

    Swift

    indirect case pushPromise(PushPromise)
  • ping(_:ack:)

    A PING frame, used to measure round-trip time between endpoints.

    See RFC 7540 § 6.7.

    Declaration

    Swift

    case ping(HTTP2PingData, ack: Bool)
  • goAway(lastStreamID:errorCode:opaqueData:)

    A GOAWAY frame, used to request that a peer immediately cease communication with the sender. It contains a stream ID indicating the last stream that will be processed by the sender, an error code (if the shutdown was caused by an error), and optionally some additional diagnostic data.

    See RFC 7540 § 6.8.

    Declaration

    Swift

    indirect case goAway(lastStreamID: HTTP2StreamID, errorCode: HTTP2ErrorCode, opaqueData: ByteBuffer?)
  • windowUpdate(windowSizeIncrement:)

    A WINDOW_UPDATE frame. This is used to implement flow control of DATA frames, allowing peers to advertise and update the amount of data they are prepared to process at any given moment.

    See RFC 7540 § 6.9.

    Declaration

    Swift

    case windowUpdate(windowSizeIncrement: Int)
  • alternativeService(origin:field:)

    An ALTSVC frame. This is sent by an HTTP server to indicate alternative origin locations for accessing the same resource, for instance via another protocol, or over TLS. It consists of an origin and a list of alternate protocols and the locations at which they may be addressed.

    See RFC 7838 § 4.

    Declaration

    Swift

    indirect case alternativeService(origin: String?, field: ByteBuffer?)
  • origin(_:)

    An ORIGIN frame. This allows servers which allow access to multiple origins via the same socket connection to identify which origins may be accessed in this manner.

    See RFC 8336 § 2.

    Swift

    case origin([String])
