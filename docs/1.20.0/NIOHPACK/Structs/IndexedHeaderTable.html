IndexedHeaderTable

public struct IndexedHeaderTable

The unified header table used by HTTP/2, encompassing both static and dynamic tables.

  • header(at:)

    Obtains the header key/value pair at the given index within the table.

    Note

    Per RFC 7541, this uses a 1-based index.

    Throws

    NIOHPACKErrors.InvalidHeaderIndex if the supplied index was invalid.

    Declaration

    Swift

    public func header(at index: Int) throws -> (name: String, value: String)

    Parameters

    index

    The index to query.

    Return Value

    A tuple containing the name and value of the stored header.

  • headerViews(at:)

    Obtains the header key/value pair at the given index within the table as sequences of raw bytes.

    Now deprecated in favour of the method that uses Strings to avoid allocations.

    Note

    Per RFC 7541, this uses a 1-based index.

    NIOHPACKErrors.InvalidHeaderIndex if the supplied index was invalid.

  • firstHeaderMatch(for:value:)

    Searches the table to locate an existing header with the given name and value. If no item exists that contains a matching value, it will return the index of the first item with a matching header name instead, to be encoded as index+value.

    Declaration

    Swift

    public func firstHeaderMatch(for name: String, value: String?) -> (index: Int, matchesValue: Bool)?

    Parameters

    name

    The name of the header to locate.
    value

    The value for which to search.

    Return Value

    A tuple containing the index of any located header, and a boolean indicating whether the item at that index also contains a matching value. Returns nil if no match could be found.

  • add(headerNamed:value:)

    Appends a header to the table.

    This call may result in an empty table, as per RFC 7541 § 4.4:

    “It is not an error to attempt to add an entry that is larger than the maximum size; an attempt to add an entry larger than the maximum size causes the table to be emptied of all existing entries and results in an empty table.”
